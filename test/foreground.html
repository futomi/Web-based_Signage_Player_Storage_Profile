Foreground fetching and storering test - Storage Profile

All of the conformant runtime must run this test appropriately.

The runtime downloads a video file as a blob object using XMLHttpRequest. Then the runtime stores the file using Indexed Database API if supported. if not, the runtime try to use File API: Directories and System.

If the file is stored successfully, the runtime gets the file as a blob object from the storage, creates a video element, then plays the video.
